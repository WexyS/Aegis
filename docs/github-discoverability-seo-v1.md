# GitHub Discoverability / SEO Alignment v1

## 1. Objective

Improve how the `WexyS/Aegis` GitHub repository presents itself to humans,
GitHub search, and external search engines without using keyword stuffing, fake
metrics, fake badges, ranking guarantees, or unsupported claims.

Aegis should be discoverable as a local-first AI control plane, a
verification-backed AI runtime, and an AI computer operator foundation with
evidence-backed execution, policy-as-code boundaries, approval-aware automation,
auditability, replayability, and bounded autonomy.

## 3. README Changes Summary

The README top section now states early that:

- Aegis is a local-first, verification-backed AI control plane and AI computer
  operator runtime.
- The repository identity is `WexyS/Aegis`.
- The differentiators are deterministic runtime truth, evidence-backed
  execution, policy-as-code boundaries, approval-aware automation, auditability,
  replayability, and bounded autonomy.

  unknown-era, replay, and resource debt.

  linked from the README.

## 9. Google Search Console Readiness

After GitHub Pages is enabled, recommended manual steps:

1. Add `https://wexys.github.io/Aegis/` as a URL-prefix property in Google Search
   Console, or use a custom domain later if one is configured.
2. Verify ownership using one of Google's supported methods.
3. Inspect `https://wexys.github.io/Aegis/` with URL Inspection.
4. Request indexing for the landing page after it returns HTTP 200.
5. Submit sitemap URL: `https://wexys.github.io/Aegis/sitemap.xml`.
6. Re-check indexing after Google has time to crawl.

The repository itself remains available at `https://github.com/WexyS/Aegis`.
Search Console setup is for the Pages surface or a future custom domain.

## 10. What Cannot Be Guaranteed

- No repository-only change can guarantee a Google ranking.
- Search indexing may take time.
- External backlinks, search behavior, repository activity, and query
  competition are outside README-only control.
- GitHub search ranking is not fully documented or controllable.
- This sprint optimizes repository clarity and discoverability legitimately; it
  does not claim guaranteed ranking improvements.
